# Ссылки в PHP (References)

> На основе [php.net/manual/ru/language.references.php](https://www.php.net/manual/ru/language.references.php). Примеры на PHP 8.x.

**Ссылка** в PHP — это способ обращаться к содержимому одной переменной под разными именами. Это **не указатель** как в C (нельзя делать арифметику адресов) — это псевдоним (alias): два имени, указывающие на одно и то же содержимое.

## Что такое ссылка

Ссылка создаётся оператором `&`. После `$b = &$a` имена `$a` и `$b` указывают на одно и то же значение — изменение через любое имя видно через другое.

```php
<?php
$a = 1;
$b = &$a;     // $b — ссылка на содержимое $a
$b = 2;
echo $a;      // 2 — изменилось через $b

// Это НЕ копия: оба имени — одна ячейка
```

Аналогия: ссылка — это не «второй ящик с копией», а «второй ярлык на том же ящике».

## Что делают ссылки

В PHP ссылки реализуют три вещи:

1. **Присваивание по ссылке** — связать две переменные.
2. **Передачу по ссылке** — функция изменяет переданную переменную.
3. **Возврат по ссылке** — функция возвращает псевдоним, а не значение.

---

## Присваивание по ссылке

```php
<?php
$original = 'значение';
$alias = &$original;       // & перед переменной-источником

$alias = 'новое';
echo $original;            // новое

// Цепочка ссылок — все указывают на одно
$x = 1;
$y = &$x;
$z = &$y;
$z = 100;
echo "$x $y $z";           // 100 100 100
```

> Важно: ссылка связывает **переменные**, а не значения. Поэтому переприсвоение `$alias = ...` меняет общее содержимое, а `unset($alias)` лишь разрывает связь, не трогая `$original`.

## Передача по ссылке (в функции)

Если параметр объявлен с `&`, функция получает доступ к исходной переменной и может её изменить.

```php
<?php
function addOne(int &$num): void
{
    $num++;                // меняет исходную переменную
}

$count = 5;
addOne($count);
echo $count;               // 6

// Классические примеры из стандартной библиотеки:
$arr = [3, 1, 2];
sort($arr);                // sort принимает массив по ссылке, меняет на месте
preg_match('/(\d+)/', 'abc123', $matches);  // $matches заполняется по ссылке
```

> ⚠️ С PHP 8.0 нельзя передавать в функцию-by-reference выражение или результат функции — только реальную переменную. Передача литерала вызовет ошибку.

## Возврат по ссылке

Функция может вернуть ссылку на свою внутреннюю переменную (применяется редко, в основном внутри классов).

```php
<?php
class Container
{
    private array $data = ['value' => 1];

    // & перед именем функции = возврат по ссылке
    public function &getValue()
    {
        return $this->data['value'];
    }
}

$c = new Container();
$ref = &$c->getValue();    // & и при вызове обязателен
$ref = 99;                 // меняет $data['value'] внутри объекта
```

> Возврат по ссылке нарушает инкапсуляцию и усложняет понимание кода — используйте только при явной необходимости.

## Ссылки и массивы

Ссылки на элементы массива ведут себя особым образом — это источник тонких багов.

```php
<?php
$arr = [1, 2, 3];
$ref = &$arr[0];           // ссылка на первый элемент
$ref = 100;
print_r($arr);             // [100, 2, 3]

// Копирование массива со ссылкой внутри сохраняет ссылку!
$a = [1, 2, 3];
$a[2] = &$a[1];            // a[2] ссылается на a[1]
$b = $a;                   // копия массива
$b[1] = 'changed';
print_r($a);              // a[2] тоже изменился! (ссылка скопировалась)
```

> При копировании массива обычные значения копируются, но **ссылки внутри массива копируются как ссылки** — это неинтуитивно.

## Ссылки и foreach (главная ловушка)

Самый частый баг с ссылками. После `foreach ($arr as &$value)` переменная `$value` остаётся ссылкой на **последний элемент** массива.

```php
<?php
$arr = [1, 2, 3];

foreach ($arr as &$value) {   // $value — ссылка на элементы
    $value *= 2;
}
// $value ВСЁ ЕЩЁ ссылается на $arr[2] (последний элемент)!

foreach ($arr as $value) {     // ❌ БАГ: первая итерация записывает в $arr[2]
    // ...
}
print_r($arr);                 // [2, 4, 4] — последний элемент испорчен!
```

**Решение — всегда делать `unset()` после foreach по ссылке:**

```php
<?php
foreach ($arr as &$value) {
    $value *= 2;
}
unset($value);                 // ✅ разорвать ссылку
```

## Ссылки и объекты

Распространённое заблуждение: «объекты в PHP передаются по ссылке». На самом деле передаётся **идентификатор объекта (handle)** по значению — это не то же самое, что ссылка `&`.

```php
<?php
class Box { public int $value = 0; }

function modify(Box $box): void {
    $box->value = 1;           // ✅ видно снаружи — общий handle
    $box = new Box();          // ❌ НЕ видно снаружи — переназначили локальную копию handle
}

$b = new Box();
modify($b);
echo $b->value;                // 1 (свойство изменилось, но $b — прежний объект)

// Чтобы переназначить сам объект — нужна настоящая ссылка:
function replace(Box &$box): void {
    $box = new Box();          // теперь видно снаружи
}
```

| | Передача объекта (без `&`) | Передача по ссылке (`&`) |
|---|----------------------------|---------------------------|
| Изменение свойств объекта | видно снаружи | видно снаружи |
| Переназначение самой переменной (`$x = new ...`) | НЕ видно | видно |

> Запомните: объект передаётся **по значению handle**, а не по ссылке. Поэтому для замены самого объекта нужен `&`.

## Уничтожение ссылок (unset)

`unset()` на ссылке разрывает связь между именем и содержимым, **не удаляя** само содержимое и другие имена.

```php
<?php
$a = 1;
$b = &$a;
unset($b);     // разрывает связь; $a остаётся
echo $a;       // 1
```

## Чего ссылки НЕ делают

- **Не являются указателями** — нельзя адресная арифметика.
- **`global $var`** внутри функции — это, по сути, ссылка на переменную в глобальной таблице.
- **`static $var`** — тоже использует механизм ссылок для сохранения значения между вызовами.
- Ссылка `$a = &$b` не ускоряет код и обычно **не экономит память** благодаря copy-on-write (PHP и так не копирует значение, пока его не изменят).

```php
<?php
// Миф: "ссылка ускорит передачу большого массива".
// Реальность: PHP использует copy-on-write — массив не копируется,
// пока его не изменят. Ссылка тут не нужна и может даже навредить
// (отключает copy-on-write оптимизацию).
function readOnly(array $big): int {   // НЕ нужен & — copy-on-write сам справится
    return count($big);
}
```

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| Что такое ссылка в PHP? | Псевдоним (alias) — несколько имён для одного содержимого. Не указатель. |
| Как создать ссылку? | Оператором `&`: `$b = &$a;`. |
| Три применения ссылок? | Присваивание, передача в функцию, возврат из функции — все через `&`. |
| Объекты передаются по ссылке? | Нет — по значению handle; свойства видны, но переназначение объекта — нет (нужен `&`). |
| Главная ловушка с `foreach`? | После `foreach (... as &$v)` нужно `unset($v)`, иначе порча последнего элемента. |
| Что делает `unset()` на ссылке? | Разрывает связь имени с содержимым, не удаляя данные. |
| Ускоряет ли ссылка передачу массива? | Нет — есть copy-on-write; ссылка может даже навредить. |
| Связаны ли `global`/`static` со ссылками? | Да — оба используют механизм ссылок внутри. |
| Можно ли передать литерал by-reference? | Нет (с 8.0) — только реальную переменную. |
