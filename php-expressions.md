# Выражения в PHP

> На основе [php.net/manual/en/language.expressions.php](https://www.php.net/manual/en/language.expressions.php). Примеры на PHP 8.x.

**Выражение (expression)** — это всё, что имеет значение. Самые базовые формы выражений — константы и переменные. Главный принцип PHP: **«почти всё является выражением»**. Каждое выражение вычисляется в значение, и это значение можно использовать дальше.

## Что такое выражение

Выражение — это конструкция, которая **вычисляется в значение**. Примеры:

```php
<?php
5                 // литерал — выражение со значением 5
$x                // переменная — выражение
$x + 1            // арифметическое выражение
foo()             // вызов функции — выражение (его значение = возврат)
$a = 3            // присваивание — тоже выражение (значение = 3)
$a > $b           // сравнение — выражение типа bool
($a > 0) ? 'да' : 'нет'  // тернарное выражение
```

Понятие выражения объясняет, почему PHP так гибок: результат одного выражения можно подставить в другое.

## Выражения и их значения

Любое выражение возвращает значение, которое можно присвоить или передать дальше.

```php
<?php
// Значение функции — это её return
function double(int $n): int { return $n * 2; }
$result = double(5);          // 10

// Выражение в выражении
$total = double(double(3));   // 12

// Значение выражения можно использовать в условии
if ($count = count($items)) { // присваивание + проверка
    echo "Элементов: $count";
}
```

## Присваивание как выражение

Оператор присваивания `=` сам является выражением — его значение равно присвоенному. Поэтому возможны цепочки и присваивания внутри условий.

```php
<?php
// Цепочка присваиваний
$a = $b = $c = 0;        // все три = 0

// Значение присваивания
$x = ($y = 5) + 3;       // $y = 5, $x = 8

// Комбинированные операторы присваивания
$n = 10;
$n += 5;   // 15  (то же, что $n = $n + 5)
$n -= 3;   // 12
$n *= 2;   // 24
$n /= 4;   // 6
$n %= 4;   // 2
$n **= 3;  // 8   (возведение в степень)
$s = 'a';
$s .= 'b'; // 'ab' (конкатенация)
$arr ??= []; // null coalescing assignment (7.4+)

// Присваивание по ссылке (это тоже выражение)
$ref = &$x;
```

## Операторы сравнения и логические

Сравнения возвращают `bool` и являются выражениями.

```php
<?php
$a == $b;    // равно (с приведением типов)
$a === $b;   // идентично (значение И тип)
$a != $b;    // не равно
$a !== $b;   // не идентично
$a <=> $b;   // spaceship: -1, 0 или 1 (PHP 7+)

// Логические операторы
$a && $b;    // И (с коротким замыканием)
$a || $b;    // ИЛИ (с коротким замыканием)
!$a;         // НЕ
$a and $b;   // И (низкий приоритет!)
$a or $b;    // ИЛИ (низкий приоритет!)
```

> **Короткое замыкание (short-circuit):** в `$a && foo()` функция `foo()` не вызывается, если `$a` ложно. Это часто используют: `$user && $user->notify();`.

> ⚠️ `and`/`or` имеют **более низкий приоритет**, чем `=`. Поэтому `$result = true and false;` присвоит `$result` значение `true` (а не `false`). Используйте `&&`/`||`.

## Тернарный оператор

Сокращённая форма `if/else` как **выражение**.

```php
<?php
// Полная форма
$status = ($age >= 18) ? 'взрослый' : 'ребёнок';

// Короткая форма (Elvis-оператор, PHP 5.3+)
$name = $input ?: 'аноним';   // = $input ? $input : 'аноним'

// ⚠️ Вложенные тернарники без скобок запрещены с PHP 8.0
// $x = $a ? 1 : $b ? 2 : 3;     // Fatal error
$x = $a ? 1 : ($b ? 2 : 3);      // нужны явные скобки
```

## Оператор null coalescing (??)

Возвращает левый операнд, если он существует и не `null`, иначе правый. Не выдаёт Warning для несуществующих ключей/переменных.

```php
<?php
// Вместо isset(...) ? ... : ...
$page = $_GET['page'] ?? 1;

// Цепочка — берётся первое не-null
$value = $a ?? $b ?? $c ?? 'default';

// Присваивание (PHP 7.4+)
$config['timeout'] ??= 30;   // установит 30, только если ключа нет/null

// Отличие от ?: (Elvis):
$x = 0;
echo $x ?? 'def';   // 0   — 0 не равно null
echo $x ?: 'def';   // def — 0 ложно
```

## Инкремент и декремент

Унарные операторы `++` и `--`, как в C. Бывают префиксные и постфиксные — это влияет на значение выражения.

```php
<?php
$i = 5;
echo $i++;   // 5, затем $i стало 6 (постфикс: вернуть, потом увеличить)
echo ++$i;   // 7 (префикс: увеличить, потом вернуть)
echo $i--;   // 7, затем 6
echo --$i;   // 5

// На строках ++ работает «по-перловски»
$s = 'Az';
$s++;        // 'Ba'
$s = 'Zz';
$s++;        // 'AAa'

// На null: ++ даёт 1, -- оставляет null
$n = null;
$n++;        // 1
```

## Выражения-конструкции

Многие конструкции PHP, которые в других языках являются операторами, в PHP **возвращают значение**:

```php
<?php
// match (PHP 8.0+) — выражение, возвращает значение, строгое сравнение (===)
$label = match ($code) {
    200, 201 => 'Success',
    404      => 'Not Found',
    default  => 'Unknown',
};

// Стрелочная функция — выражение
$double = fn ($x) => $x * 2;

// Анонимная функция (Closure) — выражение
$fn = function ($x) { return $x * 2; };

// new — выражение
$obj = new DateTime();
echo (new DateTime())->format('Y');  // можно сразу вызвать метод

// throw как выражение (PHP 8.0+)
$value = $input ?? throw new \InvalidArgumentException('нет значения');

// yield — выражение (внутри генераторов)
```

> `match` в отличие от `switch` — это выражение (возвращает значение), использует строгое сравнение `===` и не «проваливается» (нет fall-through).

## Statement vs Expression

Важно различать:

- **Выражение (expression)** — вычисляется в значение (`2 + 2`, `$x = 5`, `foo()`).
- **Инструкция/оператор (statement)** — выполняет действие, не обязательно имеет значение. Выражение становится инструкцией, если завершается `;`.

```php
<?php
$x = 5;        // выражение-инструкция (присваивание + ;)
echo "hi";     // инструкция (echo — языковая конструкция, не выражение)
if (...) {}    // управляющая инструкция (НЕ выражение)
for (...) {}   // инструкция
return $x;     // инструкция
```

> `echo`, `if`, `for`, `while`, `return` — **инструкции**, их нельзя использовать как значение. А вот `match`, тернарник, `??`, присваивание, `new`, `throw` (с 8.0) — **выражения**.

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| Что такое выражение? | Конструкция, вычисляемая в значение. |
| Является ли присваивание выражением? | Да, его значение — присвоенное (отсюда `$a = $b = 0`). |
| Разница `==` и `===`? | `==` приводит типы, `===` сравнивает значение и тип. |
| Что делает `<=>`? | Spaceship: возвращает -1/0/1. |
| Разница `&&` и `and`? | Логика та же, но `and` имеет более низкий приоритет, чем `=`. |
| Что такое короткое замыкание? | Правый операнд `&&`/`||` не вычисляется, если результат уже известен. |
| Разница `??` и `?:`? | `??` реагирует только на null/несуществование; `?:` — на любую ложность. |
| `match` vs `switch`? | `match` — выражение, строгое `===`, без fall-through. |
| Префиксный и постфиксный `++`? | Постфикс возвращает старое значение, префикс — новое. |
| `throw` — это выражение? | Да, с PHP 8.0 — можно использовать в `??`, тернарнике и т.д. |
