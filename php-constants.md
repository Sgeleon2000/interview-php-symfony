# Константы в PHP

> На основе [php.net/manual/en/language.constants.php](https://www.php.net/manual/en/language.constants.php). Примеры на PHP 8.x.

Константа — это идентификатор для значения, которое **не может измениться** во время выполнения скрипта. В отличие от переменных, перед именем константы **нет знака `$`**.

## Объявление константы

Два способа: ключевое слово `const` и функция `define()`.

```php
<?php
// Способ 1: const (на этапе компиляции)
const MAX_USERS = 100;
const APP_NAME = 'MyApp';

// Способ 2: define() (на этапе выполнения)
define('PI', 3.14159);
define('DEBUG', true);
```

По соглашению имена констант пишут в `UPPER_CASE`. Имя должно начинаться с буквы или `_`, далее буквы/цифры/`_`. Имена **регистрозависимы**.

### const vs define()

| | `const` | `define()` |
|---|---------|-----------|
| Когда обрабатывается | на этапе **компиляции** | на этапе **выполнения** |
| Можно внутри `if`/циклов | ❌ нет (только верхний уровень/класс) | ✅ да (условное объявление) |
| Имя из переменной | ❌ нет | ✅ да (`define($name, $val)`) |
| Область видимости | учитывает текущий namespace | по умолчанию глобальное |
| Значение | литерал или константное выражение | любое выражение |

```php
<?php
// define() позволяет условное объявление
if (!defined('CONFIG_LOADED')) {
    define('CONFIG_LOADED', true);
}

// define() с динамическим именем
$constName = 'DYNAMIC_' . strtoupper('flag');
define($constName, 42);
echo DYNAMIC_FLAG;   // 42

// const внутри namespace
namespace App;
const VERSION = '1.0';   // полное имя: App\VERSION
```

## Использование констант

```php
<?php
const TAX_RATE = 0.2;

$price = 100;
$total = $price * (1 + TAX_RATE);   // 120

echo APP_NAME;                       // без $

// Проверка существования
if (defined('DEBUG')) {
    echo 'DEBUG объявлена';
}

// Получить значение по имени-строке
echo constant('TAX_RATE');           // 0.2
$name = 'APP_NAME';
echo constant($name);                // динамически
```

## Особенности и ограничения

- **Глобальная видимость:** константы видны во всех областях без `global`.
- **Нельзя переопределить:** повторное `define()` той же константы выдаёт ошибку.
- **Скалярные и массивы:** значением может быть `int`, `float`, `string`, `bool`, `null`, а также **массив** (с PHP 5.6 для `const`, с 7.0 для `define()`).
- **Константное выражение:** значение может содержать другие константы и операции, вычислимые на этапе компиляции.

```php
<?php
// Массив-константа
const ROLES = ['admin', 'editor', 'user'];
define('STATUSES', ['active' => 1, 'banned' => 0]);

echo ROLES[0];               // admin
echo STATUSES['active'];     // 1

// Константное выражение
const SECONDS_IN_HOUR = 60 * 60;
const GREETING = 'Hello, ' . 'World';
const DERIVED = SECONDS_IN_HOUR * 24;  // на основе другой константы

// ❌ Нельзя: значение из вызова функции (не константное выражение)
// const NOW = time();  // Fatal error
```

> Раньше `define()` поддерживал третий аргумент `$case_insensitive` (нечувствительность к регистру) — он удалён в PHP 8.0. Константы всегда регистрозависимы.

## Магические константы

Восемь специальных констант, значение которых зависит от **места использования** (изменяются в зависимости от контекста, поэтому «магические»):

| Константа | Значение |
|-----------|----------|
| `__LINE__` | текущий номер строки |
| `__FILE__` | полный путь и имя файла |
| `__DIR__` | каталог файла (без завершающего слэша) |
| `__FUNCTION__` | имя текущей функции |
| `__CLASS__` | имя текущего класса (с namespace) |
| `__METHOD__` | имя метода (`Class::method`) |
| `__NAMESPACE__` | имя текущего пространства имён |
| `__TRAIT__` | имя текущего трейта |

```php
<?php
echo __LINE__;   // номер этой строки
echo __FILE__;   // /path/to/script.php
echo __DIR__;    // /path/to

require __DIR__ . '/config.php';   // частый приём для путей

class Service
{
    public function handle(): void
    {
        echo __CLASS__;    // Service
        echo __METHOD__;   // Service::handle
        echo __FUNCTION__; // handle
    }
}
```

> Магические константы разрешаются на этапе компиляции и пишутся в нижнем регистре с двумя подчёркиваниями с обеих сторон.

## Предопределённые константы

PHP и его расширения определяют множество констант ядра:

```php
<?php
echo PHP_VERSION;        // "8.3.0"
echo PHP_MAJOR_VERSION;  // 8
echo PHP_EOL;            // перевод строки текущей ОС
echo PHP_INT_MAX;        // максимальное int
echo PHP_INT_SIZE;       // размер int в байтах (8 на 64-бит)
echo PHP_FLOAT_EPSILON;  // погрешность float
echo DIRECTORY_SEPARATOR;// / или \
echo PHP_OS;             // "Linux", "Darwin", "WINNT"

// Логические/специальные
var_dump(true, false, null);
echo M_PI;               // 3.14159... (math)
echo E_ALL;              // битовая маска уровней ошибок
```

## Константы классов

Объявляются через `const` внутри класса/интерфейса. Доступны через `::`. По умолчанию `public`, но можно задать видимость (PHP 7.1+).

```php
<?php
class HttpStatus
{
    public const OK = 200;
    public const NOT_FOUND = 404;
    protected const INTERNAL = 'secret';
    final public const VERSION = '1.1';   // final-константа (8.1+) — нельзя переопределить

    public function describe(): string
    {
        return self::OK . ' OK';   // обращение внутри класса
    }
}

echo HttpStatus::OK;               // 200
echo HttpStatus::class;            // "HttpStatus" — спец. константа ::class

// Доступ через объект тоже работает
$s = new HttpStatus();
echo $s::NOT_FOUND;                // 404

// Интерфейсные константы
interface Logger {
    const LEVEL_DEBUG = 'debug';
}
```

Особенности констант классов:
- `self::CONST` — раннее связывание (по текущему классу).
- `static::CONST` — позднее статическое связывание (с учётом наследника).
- `parent::CONST` — константа родителя.
- `ClassName::class` — полное имя класса в виде строки.
- Типизированные константы классов появились в **PHP 8.3**: `const int OK = 200;`.

## Enum как набор констант

С PHP 8.1 для фиксированного набора значений лучше подходят **перечисления** (enum) — они типобезопаснее набора констант.

```php
<?php
// Вместо набора констант:
// const STATUS_DRAFT = 'draft'; const STATUS_PUBLISHED = 'published';

enum Status: string
{
    case Draft = 'draft';
    case Published = 'published';
    case Archived = 'archived';
}

function publish(Status $status): void { /* типобезопасно */ }
publish(Status::Published);   // нельзя передать случайную строку
```

> Подробнее об enum — см. `php-types.md`.

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| Чем константа отличается от переменной? | Не меняется, без `$`, глобально видима. |
| `const` vs `define()`? | `const` — компиляция, верхний уровень; `define()` — выполнение, можно условно и с динамическим именем. |
| Может ли значение быть массивом? | Да (`const` с 5.6, `define()` с 7.0). |
| Что такое магические константы? | `__LINE__`, `__FILE__`, `__DIR__`, `__CLASS__` и т.д. — зависят от места использования. |
| Как проверить существование константы? | `defined('NAME')`. |
| Как получить значение по строке-имени? | `constant('NAME')`. |
| Видимость констант класса? | По умолчанию `public`; с 7.1 можно `protected`/`private`; с 8.1 — `final`. |
| `self::` vs `static::` для констант? | `self` — текущий класс; `static` — позднее статическое связывание. |
| Что лучше для набора значений в PHP 8.1+? | `enum` — типобезопаснее набора констант. |
