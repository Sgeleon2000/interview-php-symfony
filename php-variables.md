# Переменные в PHP

> На основе [php.net/manual/en/language.variables.php](https://www.php.net/manual/en/language.variables.php). Примеры на PHP 8.x.

Переменная в PHP — это именованная ссылка на значение. Тип определяется значением (динамическая типизация), объявлять переменную заранее не нужно — она «появляется» в момент присваивания.

## Основы

Имя переменной начинается с `$`, далее буква или `_`, затем буквы, цифры, `_`. Имена **регистрозависимы**.

```php
<?php
$name = 'Анна';     // строка
$age = 30;          // int
$_private = true;   // подчёркивание допустимо в начале
$имя = 'Кириллица'; // допустимо, но не рекомендуется

echo $name;         // Анна
echo $Name ?? 'нет';// $Name — другая переменная (регистр важен)
```

Переменная без присваивания не определена; чтение выдаёт **Warning** и возвращает `null`.

```php
<?php
echo $undefined;          // Warning: Undefined variable, выводит null
echo $undefined ?? 'def'; // безопасно: def
```

## Присваивание по значению и по ссылке

По умолчанию присваивание идёт **по значению** — копия данных. Чтобы связать две переменные с одной ячейкой памяти, используют **ссылку** (`&`).

```php
<?php
// По значению
$a = 10;
$b = $a;
$b = 20;
echo $a;   // 10 — $a не изменилась

// По ссылке
$x = 10;
$y = &$x;  // $y ссылается на ту же ячейку
$y = 20;
echo $x;   // 20 — изменилась и $x

unset($y); // разрывает ссылку, $x остаётся 20
```

> **Объекты — особый случай.** При присваивании объекта копируется не сам объект, а его *идентификатор* (handle). Поэтому изменение через одну переменную видно через другую — но это **не** ссылка (`unset` одной не влияет на другую). Для полной копии используйте `clone`.

```php
<?php
$o1 = new stdClass();
$o1->v = 1;
$o2 = $o1;       // копируется handle, не объект
$o2->v = 99;
echo $o1->v;     // 99

$o3 = clone $o1; // независимая копия
$o3->v = 5;
echo $o1->v;     // 99
```

## Область видимости (scope)

По умолчанию переменная видна только в том контексте, где объявлена. Внутри функции **не видны** переменные снаружи (в отличие от многих языков — нет автоматического доступа к внешнему scope).

```php
<?php
$outer = 'снаружи';

function test() {
    echo $outer ?? 'недоступна'; // недоступна — другой scope
}
test();
```

### Глобальные переменные и global

Чтобы обратиться к глобальной переменной внутри функции — ключевое слово `global` или массив `$GLOBALS`.

```php
<?php
$counter = 0;

function increment() {
    global $counter;   // привязка к глобальной
    $counter++;
}
// эквивалент:
function increment2() {
    $GLOBALS['counter']++;
}

increment();
echo $counter;         // 1
```

> Глобальные переменные усложняют тестирование и связывают код. Предпочтительнее передавать значения через **аргументы** или **DI**.

### Статические переменные

Статическая переменная сохраняет значение **между вызовами** функции, но видна только внутри неё. Инициализируется один раз.

```php
<?php
function counter(): int {
    static $count = 0;  // инициализируется один раз
    return ++$count;
}

echo counter(); // 1
echo counter(); // 2
echo counter(); // 3
```

Часто используется для мемоизации и ленивой инициализации.

## Предопределённые переменные (суперглобалы)

**Суперглобальные** массивы доступны в любой области видимости без `global`:

| Переменная | Содержимое |
|-----------|-----------|
| `$GLOBALS` | все переменные глобальной области |
| `$_SERVER` | данные сервера и окружения выполнения |
| `$_GET` | параметры строки запроса (`?a=1`) |
| `$_POST` | данные POST-запроса |
| `$_REQUEST` | объединение `$_GET`, `$_POST`, `$_COOKIE` |
| `$_FILES` | загруженные файлы |
| `$_COOKIE` | HTTP cookies |
| `$_SESSION` | переменные сессии |
| `$_ENV` | переменные окружения |

```php
<?php
$page   = $_GET['page'] ?? 1;
$method = $_SERVER['REQUEST_METHOD'];
$host   = $_SERVER['HTTP_HOST'] ?? 'localhost';

// ⚠️ Данные из $_GET/$_POST/$_REQUEST/$_COOKIE — небезопасны.
// Всегда валидируйте и экранируйте (защита от XSS, SQL-инъекций).
$id = filter_input(INPUT_GET, 'id', FILTER_VALIDATE_INT);
```

## Переменные переменных (variable variables)

Имя переменной можно получить из значения другой переменной — через `$$`.

```php
<?php
$varName = 'color';
$$varName = 'красный';   // создаёт переменную $color

echo $color;             // красный
echo ${$varName};        // красный (явная форма, безопаснее)

// С массивами нужны фигурные скобки для устранения неоднозначности
$key = 'items';
${$key}[] = 'value';     // $items[] = 'value'
```

> Применяется редко (динамическая генерация имён); снижает читаемость. Чаще лучше использовать обычный массив.

## Переменные из внешних источников

Данные HTML-форм, URL и cookie автоматически попадают в суперглобалы.

```php
<?php
// Форма: <input name="user"> и <input name="prefs[]">
$user  = $_POST['user'] ?? '';
$prefs = $_POST['prefs'] ?? [];  // массив для name="prefs[]"

// Точки/пробелы в именах полей заменяются на _:
// поле "user.name" → $_POST['user_name']
```

Старые механизмы автоматического создания переменных из запроса (**`register_globals`**) удалены ещё в PHP 5.4 как небезопасные — внешние данные доступны только через суперглобалы.

## Проверка и удаление переменных

```php
<?php
$a = 0;
$b = null;

var_dump(isset($a));        // true  — установлена и не null
var_dump(isset($b));        // false — null считается «не установленной»
var_dump(empty($a));        // true  — 0 считается «пустым»
var_dump(empty('text'));    // false

// Объединённые проверки
$value = $config['key'] ?? 'default';        // null coalescing
$config['key'] ??= 'default';                 // null coalescing assignment (7.4+)

unset($a);                  // удалить переменную
var_dump(isset($a));        // false
```

**`empty()` считает «пустыми»:** `""`, `0`, `0.0`, `"0"`, `null`, `false`, `[]`, неустановленную переменную.

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| Как объявить переменную? | Просто присвоить значение: `$x = 1;` — тип определит значение. |
| Регистрозависимы ли имена? | Да: `$name` и `$Name` — разные. |
| Разница присваивания по значению и по ссылке? | По значению — копия; по ссылке (`&`) — общая ячейка памяти. |
| Как объекты передаются при присваивании? | Копируется handle (идентификатор), не данные; для копии — `clone`. |
| Видны ли внешние переменные внутри функции? | Нет, нужен `global` или `$GLOBALS`. |
| Что такое static-переменная? | Сохраняет значение между вызовами функции, инициализируется один раз. |
| Что такое суперглобалы? | Массивы (`$_GET`, `$_POST`, `$_SERVER`…), доступные везде без `global`. |
| Разница `isset()` и `empty()`? | `isset` — установлена и не null; `empty` — «пустое» значение (0, "", [] и т.д.). |
| Что такое переменные переменных? | `$$x` — имя переменной берётся из значения другой (`$$x`). |
| Чем опасны данные из `$_GET`/`$_POST`? | Это внешний ввод — нужны валидация и экранирование. |
