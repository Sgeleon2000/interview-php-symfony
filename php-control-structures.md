# Управляющие конструкции в PHP

> На основе [php.net/manual/en/language.control-structures.php](https://www.php.net/manual/en/language.control-structures.php). Примеры на PHP 8.x.

Управляющие конструкции определяют **порядок выполнения** кода: ветвление, циклы, прерывания, подключение файлов. Любой PHP-скрипт — это последовательность инструкций, а управляющие конструкции меняют линейный поток.

## Условные конструкции

### if / elseif / else

Базовое ветвление. Выполняет блок, если выражение истинно.

```php
<?php
$score = 75;

if ($score >= 90) {
    echo 'Отлично';
} elseif ($score >= 60) {     // слитно: elseif (или раздельно else if)
    echo 'Хорошо';
} else {
    echo 'Плохо';
}

// Однострочно без скобок (только одна инструкция)
if ($x > 0) echo 'положительное';
```

> Условие приводится к `bool` по правилам PHP: `0`, `""`, `"0"`, `[]`, `null`, `0.0`, `false` → false.

### switch

Сравнивает значение с набором вариантов. Использует **нестрогое сравнение (`==`)** и имеет «проваливание» (fall-through) — без `break` выполнение продолжается в следующий `case`.

```php
<?php
switch ($role) {
    case 'admin':
    case 'moderator':           // несколько case подряд — общий блок
        echo 'Есть доступ';
        break;                  // обязателен, иначе провалится дальше
    case 'user':
        echo 'Ограниченный доступ';
        break;
    default:
        echo 'Неизвестная роль';
}
```

> ⚠️ `switch` использует `==`: `case "1":` совпадёт со значением `1`. Забытый `break` — частый источник багов.

### match (PHP 8.0+)

Современная альтернатива `switch`: это **выражение** (возвращает значение), использует **строгое сравнение (`===`)** и **не проваливается**.

```php
<?php
$result = match ($status) {
    200, 201, 204 => 'Success',     // несколько значений через запятую
    301, 302      => 'Redirect',
    404           => 'Not Found',
    default       => 'Unknown',
};

// match без аргумента-условия (проверка произвольных выражений)
$category = match (true) {
    $age < 13  => 'ребёнок',
    $age < 18  => 'подросток',
    default    => 'взрослый',
};

// Без default бросает UnhandledMatchError, если совпадений нет
```

| | `switch` | `match` |
|---|----------|---------|
| Сравнение | `==` (нестрогое) | `===` (строгое) |
| Возвращает значение | ❌ | ✅ выражение |
| Fall-through | ✅ (нужен `break`) | ❌ |
| Нет совпадения | тихо пропускает | `UnhandledMatchError` |

## Циклы

### while

Выполняет тело, **пока** условие истинно. Проверка — перед каждой итерацией.

```php
<?php
$i = 1;
while ($i <= 5) {
    echo $i;
    $i++;
}   // 12345
```

### do-while

Как `while`, но условие проверяется **после** тела — гарантирует хотя бы одно выполнение.

```php
<?php
$i = 10;
do {
    echo $i;          // выполнится 1 раз, даже если условие ложно
    $i++;
} while ($i < 5);     // выводит 10
```

### for

Цикл со счётчиком: `for (инициализация; условие; шаг)`.

```php
<?php
for ($i = 0; $i < 5; $i++) {
    echo $i;          // 01234
}

// Несколько переменных, любые части можно опустить
for ($i = 0, $j = 10; $i < $j; $i++, $j--) {
    echo "$i-$j ";
}
```

### foreach

Главный цикл для обхода массивов и объектов (`Traversable`). Два варианта:

```php
<?php
$colors = ['red', 'green', 'blue'];

// Только значения
foreach ($colors as $color) {
    echo $color;
}

// Ключ => значение
$user = ['name' => 'Анна', 'age' => 30];
foreach ($user as $key => $value) {
    echo "$key: $value\n";
}

// Изменение элементов по ссылке
foreach ($colors as &$color) {
    $color = strtoupper($color);
}
unset($color);   // ВАЖНО: разорвать ссылку после цикла!

// Деструктуризация (вложенные массивы)
$points = [[1, 2], [3, 4]];
foreach ($points as [$x, $y]) {
    echo "($x, $y) ";
}
```

> ⚠️ После `foreach (... as &$ref)` обязательно делайте `unset($ref)` — иначе последний элемент может быть случайно перезаписан в дальнейшем коде.

## Управление циклом

### break / continue

- **`break`** — немедленно выходит из цикла (или `switch`).
- **`continue`** — пропускает оставшуюся часть текущей итерации, переходит к следующей.

Оба принимают **необязательный числовой аргумент** — сколько уровней вложенных циклов прервать/пропустить.

```php
<?php
for ($i = 0; $i < 10; $i++) {
    if ($i === 5) break;       // выйти из цикла при 5
    if ($i % 2 === 0) continue;// пропустить чётные
    echo $i;                   // 1 3
}

// Многоуровневый break
foreach ($rows as $row) {
    foreach ($row as $cell) {
        if ($cell === null) {
            break 2;           // выйти сразу из обоих циклов
        }
    }
}
```

## Альтернативный синтаксис

Для каждой конструкции есть форма с двоеточием и `endif`/`endwhile`/`endfor`/`endforeach`/`endswitch`. Удобна при смешивании PHP и HTML в шаблонах.

```php
<?php if ($loggedIn): ?>
    <p>Привет, <?= $name ?></p>
<?php elseif ($guest): ?>
    <p>Гость</p>
<?php else: ?>
    <p>Войдите</p>
<?php endif; ?>

<ul>
<?php foreach ($items as $item): ?>
    <li><?= htmlspecialchars($item) ?></li>
<?php endforeach; ?>
</ul>
```

> Этот синтаксис широко применялся в шаблонах; в современных проектах его заменяют движки (Twig, Blade).

## Прочие конструкции

### declare

Задаёт директивы выполнения для блока/файла. Самая частая — `strict_types`.

```php
<?php
declare(strict_types=1);   // строгая типизация; ДОЛЖНА быть первой инструкцией

declare(ticks=1);          // тики (устаревающий механизм)
```

### return / goto

```php
<?php
// return — возврат из функции (со значением) или завершение скрипта на верхнем уровне
function sum(int $a, int $b): int {
    return $a + $b;
}

// goto — переход к метке (использовать крайне редко, ухудшает читаемость)
for ($i = 0; $i < 10; $i++) {
    if ($i === 3) goto end;
}
end:
echo 'Готово';
```

### include / require

Подключают и выполняют другой PHP-файл.

| Конструкция | Если файл не найден |
|-------------|---------------------|
| `include` | **Warning**, скрипт продолжается |
| `require` | **Fatal error**, скрипт останавливается |
| `include_once` | то же + не подключит повторно |
| `require_once` | то же + не подключит повторно |

```php
<?php
require __DIR__ . '/vendor/autoload.php';  // критично — require
include 'optional-banner.php';             // необязательно — include

require_once 'config.php';                 // защита от повторного подключения

// include/require возвращают значение, если в файле есть return
$config = require 'config.php';            // файл: <?php return ['db' => ...];
```

> На практике ручные `include`/`require` для классов заменены **автозагрузкой** (PSR-4 через Composer) — см. `psr-standards.md`.

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| `switch` vs `match`? | `match` — выражение, строгое `===`, без fall-through, бросает ошибку без совпадения. |
| Чем `while` отличается от `do-while`? | `do-while` выполняет тело минимум один раз (условие после). |
| Когда использовать `foreach as &$v`? | Для изменения элементов; обязательно `unset($v)` после. |
| Что делает `break 2`? | Выходит сразу из двух вложенных циклов. |
| `continue` в цикле — что делает? | Пропускает остаток итерации, переходит к следующей. |
| `include` vs `require`? | `require` при отсутствии файла — Fatal error; `include` — Warning. |
| Зачем `*_once`? | Предотвращает повторное подключение того же файла. |
| Где должен стоять `declare(strict_types=1)`? | Первой инструкцией файла. |
| Что такое альтернативный синтаксис? | Формы `if: ... endif;` и т.п. для шаблонов с HTML. |
| Использовать ли `goto`? | Крайне редко — ухудшает читаемость. |
