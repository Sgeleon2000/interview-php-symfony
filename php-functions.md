# Функции в PHP

> На основе [php.net/manual/en/language.functions.php](https://www.php.net/manual/en/language.functions.php). Примеры на PHP 8.x.

Функция — это именованный блок кода, который можно вызывать многократно. PHP поддерживает пользовательские функции, аргументы (в т.ч. по умолчанию, по ссылке, именованные, вариативные), возвращаемые значения, анонимные функции, замыкания и стрелочные функции.

## Объявление функции

```php
<?php
function greet(string $name): string
{
    return "Привет, $name!";
}

echo greet('Анна');   // Привет, Анна!
```

Особенности:
- Имена функций **регистронезависимы** (`greet()` == `Greet()`), но по соглашению пишут `camelCase` или `snake_case`.
- Функция, объявленная на верхнем уровне (не внутри условия/функции), доступна **во всём скрипте** ещё до строки объявления (hoisting).
- Условно объявленную функцию (внутри `if`) можно вызвать только **после** выполнения условия.

```php
<?php
echo early();   // работает — функция «поднята»
function early(): string { return 'ok'; }

if (true) {
    function conditional() { return 'cond'; }
}
echo conditional();  // работает только после выполнения if
```

## Аргументы

### Значения по умолчанию

Параметры со значением по умолчанию должны идти **после** обязательных.

```php
<?php
function makeCoffee(string $type = 'эспрессо', int $sugar = 0): string
{
    return "$type, сахара: $sugar";
}

echo makeCoffee();              // эспрессо, сахара: 0
echo makeCoffee('латте');       // латте, сахара: 0
echo makeCoffee('латте', 2);    // латте, сахара: 2

// Значение по умолчанию — константное выражение
function setup(array $opts = ['debug' => false]) {}
```

### Передача по ссылке

По умолчанию аргументы передаются **по значению** (копия). Знак `&` перед параметром передаёт по ссылке — функция может изменить исходную переменную.

```php
<?php
function addOne(int &$num): void
{
    $num++;
}

$x = 5;
addOne($x);
echo $x;   // 6 — изменилась исходная переменная

// Часто встречается в стандартной библиотеке: sort(), array_push() и др.
$arr = [3, 1, 2];
sort($arr);   // изменяет $arr на месте
```

> Объекты «и так» передаются по handle — изменение свойств видно снаружи без `&`. Но `&` нужен, чтобы переназначить саму переменную внутри функции.

### Вариативные функции (...)

Оператор `...` позволяет принять **произвольное число аргументов** в массив.

```php
<?php
function sum(int ...$numbers): int
{
    return array_sum($numbers);   // $numbers — массив
}

echo sum(1, 2, 3, 4);   // 10
echo sum();             // 0

// Распаковка массива в аргументы (splat) при вызове
$nums = [1, 2, 3];
echo sum(...$nums);     // 6

// Комбинирование обычных и вариативных
function format(string $sep, string ...$parts): string
{
    return implode($sep, $parts);
}
echo format('-', 'a', 'b', 'c');   // a-b-c
```

### Именованные аргументы (PHP 8.0+)

Можно передавать аргументы по имени параметра — порядок не важен, удобно пропускать необязательные.

```php
<?php
function createUser(string $name, bool $admin = false, bool $active = true): void {}

// Позиционные + именованные
createUser('Анна', active: false);          // пропустили $admin
createUser(name: 'Иван', admin: true);

// Особенно полезно для функций с множеством флагов
htmlspecialchars($str, double_encode: false);
```

> Именованные аргументы делают вызовы читаемее и устойчивее к изменению порядка параметров. Позиционные должны идти **перед** именованными.

### Типизация аргументов

```php
<?php
function process(
    int $id,                       // скалярный тип
    ?string $note,                 // nullable
    int|float $amount,             // union (8.0+)
    array $tags = [],              // составной тип
    callable $callback = null,     // вызываемое
): void {}

// При declare(strict_types=1) несоответствие типа → TypeError
// Иначе PHP пытается привести тип (int → float и т.п.)
```

## Возвращаемые значения

Функция возвращает значение через `return`. Без `return` (или с пустым) возвращается `null`.

```php
<?php
function divide(int $a, int $b): float
{
    if ($b === 0) {
        throw new \InvalidArgumentException('Деление на ноль');
    }
    return $a / $b;
}

// Возврат нескольких значений — через массив + деструктуризация
function minMax(array $nums): array
{
    return [min($nums), max($nums)];
}
[$lo, $hi] = minMax([3, 1, 4, 1, 5]);

// Возврат по ссылке (редко): function &getRef() { ... }

// void — функция ничего не возвращает; never — никогда не вернётся
function log(string $m): void { /* ... */ }
function fail(): never { throw new \RuntimeException(); }
```

## Анонимные функции (Closure)

Функция без имени, которую можно присвоить переменной или передать как аргумент. Это объекты класса `Closure`.

```php
<?php
$multiply = function (int $a, int $b): int {
    return $a * $b;
};
echo $multiply(3, 4);   // 12

// Захват переменных из внешней области через use
$factor = 10;
$scale = function (int $x) use ($factor): int {
    return $x * $factor;
};
echo $scale(5);   // 50

// Захват по ссылке через use (&...)
$counter = 0;
$increment = function () use (&$counter): void {
    $counter++;
};
$increment();
echo $counter;   // 1

// Типичное применение — колбэки
$even = array_filter([1, 2, 3, 4], fn($n) => $n % 2 === 0);
```

> Важное отличие от стрелочных функций: обычная анонимная функция **не видит** внешние переменные автоматически — их нужно явно перечислить в `use`.

## Стрелочные функции (PHP 7.4+)

Краткая форма анонимной функции. **Автоматически** захватывает переменные из окружения (по значению), тело — одно выражение.

```php
<?php
$factor = 3;
$scale = fn(int $x): int => $x * $factor;   // $factor захвачена автоматически
echo $scale(5);   // 15

// Идеальны для коротких колбэков
$names = array_map(fn($u) => $u->name, $users);
$sorted = usort($items, fn($a, $b) => $a->price <=> $b->price);
```

| | Анонимная функция | Стрелочная функция |
|---|-------------------|--------------------|
| Синтаксис | `function () use (...) {}` | `fn() =>` |
| Захват переменных | явно через `use` | автоматически (по значению) |
| Тело | любое число инструкций | одно выражение |
| Захват по ссылке | да (`use (&$x)`) | нет |

## Callable и first-class callable

Функцию можно передавать разными способами:

```php
<?php
// Имя функции строкой
$f = 'strlen';
echo $f('hello');   // 5

// Метод объекта / статический метод
$callable1 = [$object, 'methodName'];
$callable2 = ['ClassName', 'staticMethod'];
$callable3 = 'ClassName::staticMethod';

// First-class callable syntax (PHP 8.1+) — лучший способ
$strlen = strlen(...);
$method = $object->method(...);
$static = ClassName::staticMethod(...);

echo $strlen('hi');   // 2

// Проверка вызываемости
var_dump(is_callable('strlen'));   // true
```

## Рекурсия и вложенные функции

```php
<?php
// Рекурсия
function factorial(int $n): int
{
    return $n <= 1 ? 1 : $n * factorial($n - 1);
}
echo factorial(5);   // 120

// Вложенная функция — становится глобальной после вызова внешней
function outer(): void
{
    function inner(): string { return 'inner'; }  // объявляется при вызове outer()
}
outer();
echo inner();   // 'inner' — теперь доступна глобально
```

> Глубокая рекурсия может упереться в лимит стека. Для самовызова анонимных функций используйте `use (&$self)`.

## Особенности области видимости

- Внутри функции **не видны** глобальные переменные (нужен `global` или `$GLOBALS`) — см. `php-variables.md`.
- Параметры и локальные переменные существуют только во время вызова.
- `static`-переменные внутри функции сохраняют значение между вызовами.

```php
<?php
function idGenerator(): int
{
    static $id = 0;   // сохраняется между вызовами
    return ++$id;
}
echo idGenerator();   // 1
echo idGenerator();   // 2
```

## Шпаргалка для собеседования

| Вопрос | Ответ |
|--------|-------|
| Регистрозависимы ли имена функций? | Нет (но имена классов/переменных — да). |
| Как передать аргумент по ссылке? | `function f(&$x)`. |
| Что такое вариативная функция? | `function f(...$args)` — принимает любое число аргументов в массив. |
| Что такое splat-оператор при вызове? | `f(...$array)` — распаковка массива в аргументы. |
| Зачем именованные аргументы? | Читаемость и пропуск необязательных параметров (8.0+). |
| Closure vs arrow function? | Closure — захват через `use`, многострочный; arrow — авто-захват, одно выражение. |
| Захватывает ли `fn` по ссылке? | Нет, только по значению. |
| Как вернуть несколько значений? | Через массив + деструктуризацию `[$a, $b] = f()`. |
| Что такое first-class callable? | `strlen(...)` — ссылка на функцию как Closure (8.1+). |
| Что делает `static` внутри функции? | Сохраняет значение переменной между вызовами. |
| Чем `void` отличается от `never`? | `void` — ничего не возвращает; `never` — никогда не завершается нормально. |
